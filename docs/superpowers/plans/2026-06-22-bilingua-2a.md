# Bilingua 2a — Static UI & Admin → English-primary — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make English the primary language of the public site and the Django admin, with Ukrainian as a switchable variant, by moving to English source strings (classic gettext) plus a `locale/uk` catalog.

**Architecture:** Flip i18n settings to English first (delivers English-primary immediately via the existing `en.po`), then refactor source strings from Ukrainian to English layer-by-layer, build `locale/uk` by inverting the existing `en.po`, delete `locale/en`. `SiteSettings` (the only modeltranslation model) gets English field defaults, a default-language flip, and `brand_name` becomes a single (non-translated) field.

**Tech Stack:** Django 5.2, django-modeltranslation, gettext (`makemessages`/`compilemessages`), `polib` (one-time dev helper only — NOT a shipped dependency).

## Global Constraints

- **Source language = English.** After this plan, code string literals in `_()`, `{% trans %}`, `{% blocktrans %}`, `verbose_name`, `help_text`, choices labels, form labels/placeholders/errors, validator messages, and Jazzmin branding are **English**. Ukrainian lives only in `locale/uk/LC_MESSAGES/django.po`.
- **`en.po` is the authoritative English source-of-record.** For every Ukrainian literal being rewritten, the English to use is that literal's `msgstr` in `locale/en/LC_MESSAGES/django.po` (msgid = Ukrainian, msgstr = English). The 9 strings with empty `msgstr` are listed explicitly in Task 7.
- **`brand_name` is the site-name exception:** it is removed from translation and becomes a single field.
- **Fallback:** `MODELTRANSLATION_FALLBACK_LANGUAGES = ("en", "uk")` so neither language renders blank.
- **No new shipped dependency.** `polib` is installed ad-hoc in the container for the one-time `uk.po` build; the committed `uk.po` is the artifact. Do not add `polib` to `requirements.txt`.
- **modeltranslation invariant:** the base field always mirrors the default-language field; after flipping default to `en`, `X` (base) must equal `X_en`.
- **Code comments / docstrings stay Ukrainian** — out of scope (not user-facing). Cyrillic gates below target string literals only, not comments.
- **Tests run in Docker:** `docker compose -f docker-compose.dev.yml exec -T web python manage.py <cmd> --settings=config.settings.development` (run from `d:\retro`; container module paths use app labels, no `distributor.` prefix). The dev stack is already running — do not rebuild.

**Current state:** `LANGUAGE_CODE="uk"`, MT default `uk`, fallback `("uk","en")`; only `locale/en` (354 msgids, 9 untranslated, 16 obsolete, 1 plural, 0 fuzzy/context); `apps/core/translation.py` registers 24 `SiteSettings` fields incl. `brand_name`. Branch for this work: `bilingua-2a` (already checked out).

```python
from django.db import migrations

TRANSLATED = [
    "tagline", "meta_description", "footer_copyright", "cta_label",
    "nav_catalog_label", "nav_brands_label", "nav_promos_label",
    "nav_news_label", "nav_vacancies_label", "nav_contacts_label",
    "term_product_singular", "term_product_plural",
    "term_brand_singular", "term_brand_plural",
    "term_category_singular", "term_category_plural",
    "vacancy_description_label", "vacancy_requirements_label",
    "vacancy_conditions_label", "vacancy_apply_label",
    "hero_eyebrow", "hero_title", "hero_subtitle",
]


def seed_english(apps, schema_editor):
    SiteSettings = apps.get_model("core", "SiteSettings")
    obj = SiteSettings.objects.first()
    if obj is None:
        return
    for name in TRANSLATED:
        en = getattr(obj, f"{name}_en", "") or ""
        uk = getattr(obj, f"{name}_uk", "") or ""
        base = getattr(obj, name, "") or ""
        # Existing data was authored under uk-default, so base == uk value.
        if not uk:
            uk = base
        if not en:
            field = SiteSettings._meta.get_field(f"{name}_en")
            en = field.default if field.has_default() and isinstance(field.default, str) else ""
        setattr(obj, f"{name}_uk", uk)
        setattr(obj, f"{name}_en", en)
        setattr(obj, name, en)  # base mirrors default language (en)
    obj.save()


class Migration(migrations.Migration):
    dependencies = [("core", "0013_<replace_with_generated_name>")]
    operations = [migrations.RunPython(seed_english, migrations.RunPython.noop)]
```
